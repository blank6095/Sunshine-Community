## 设计原则

### 1. 模块清晰原则
- 按功能模块划分而非技术层次
- 相关文件放置在一起（High Cohesion）
- 模块间保持低耦合（Low Coupling）

### 2. 可扩展性原则
- 预留插件扩展目录
- 组件库独立管理
- 配置文件集中管理

### 3. 可维护性原则
- 每个文件职责单一
- 完善的类型定义
- 统一的代码规范

### 4. 团队协作原则
- 清晰的目录划分
- 完整的文档说明
- 可复用的组件库

## Vue3 vs UniApp 架构差异

### Vue3 独立项目架构特点

```
vue3-project/
├── src/
│   ├── router/          # Vue Router 配置
│   ├── views/          # 页面组件
│   ├── public/         # 公共资源
│   └── index.html      # 入口HTML
├── vite.config.js
└── package.json
```

**特点**：
- 使用 Vue Router 管理路由
- 完整的 SPA 应用
- 灵活的组件导入
- 仅支持 Web 端

---

### UniApp 跨平台架构特点

```
uniapp-project/
├── src/
│   ├── pages/          # 页面目录（必须）
│   ├── static/        # 静态资源（必须）
│   ├── App.vue       # 应用入口（必须）
│   ├── main.ts       # 主文件（必须）
│   ├── pages.json    # 路由配置（必须）
│   └── manifest.json # 应用配置（必须）
└── vite.config.js
```

**特点**：
- 使用 pages.json 配置路由
- 支持多端（H5、小程序、App）
- 组件和页面统一管理
- 目录结构有严格要求

---

### 核心差异对比表

| 维度 | Vue3 独立项目 | UniApp 跨平台项目 |
|------|--------------|------------------|
| **路由管理** | Vue Router | pages.json |
| **入口文件** | index.html + main.js | App.vue + main.ts + pages.json |
| **页面注册** | router/index.js | pages.json |
| **静态资源** | public/ | static/ |
| **组件存放** | components/ | components/ |
| **状态管理** | Pinia/Vuex | Pinia |
| **API 调用** | axios/fetch | uni.request |
| **存储** | localStorage | uni.setStorage |
| **跳转方式** | router.push() | uni.navigateTo() |
| **目标平台** | 仅 Web | H5、小程序、App 多端 |
| **编译工具** | Vite | Vite + UniApp 编译器 |
| **样式隔离** | scoped | 自动隔离 |
| **平台判断** | window | uni.getSystemInfoSync() |
