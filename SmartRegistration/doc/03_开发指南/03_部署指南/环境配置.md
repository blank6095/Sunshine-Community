# 环境配置

## 1. 概述

本文档详细说明智能挂号预约系统的环境配置，帮助运维人员了解如何配置和管理系统的运行环境。

## 2. 配置文件管理

### 2.1 配置文件结构

系统使用以下配置文件：

| 配置文件 | 用途 | 环境 |
|----------|------|------|
| application.yml | 主配置文件 | 通用 |
| application-dev.properties | 开发环境配置 | 开发 |
| application-test.properties | 测试环境配置 | 测试 |
| application-prod.properties | 生产环境配置 | 生产 |

### 2.2 配置文件优先级

Spring Boot的配置文件优先级（从高到低）：

1. 命令行参数
2. 环境变量
3. application-{profile}.properties/yml
4. application.properties/yml
5. 内置的默认配置

## 3. 核心配置项

### 3.1 服务器配置

```yaml
# 服务器配置
server:
  port: 8080
  servlet:
    context-path: /api/v1
  tomcat:
    max-threads: 200
    min-spare-threads: 20
    connection-timeout: 60000
```

### 3.2 数据库配置

```yaml
# 数据库配置
spring:
  datasource:
    url: jdbc:mysql://localhost:3306/smart_registration_db?useSSL=false&serverTimezone=UTC&useLegacyDatetimeCode=false
    username: root
    password: 123456
    driver-class-name: com.mysql.cj.jdbc.Driver
    hikari:
      maximum-pool-size: 20
      minimum-idle: 5
      idle-timeout: 30000
      connection-timeout: 30000
      max-lifetime: 120000

  # JPA配置
  jpa:
    hibernate:
      ddl-auto: update
    show-sql: true
    properties:
      hibernate:
        format_sql: true
        dialect: org.hibernate.dialect.MySQL8Dialect
```

### 3.3 JWT配置

```yaml
# JWT配置
jwt:
  secret: your-secret-key-here
  expiration:
    default: 86400
    unit: seconds
    access-token: 3600
    refresh-token: 604800
```

### 3.4 认证豁免配置

```yaml
# 认证豁免配置
auth:
  exemption:
    enable-logging: true
    enable-dynamic-reload: true
    rules:
      - path: /auth/login
        http-methods:
          - POST
        description: 登录接口豁免认证
        pattern-type: EXACT
        enabled: true
      - path: /auth/register
        http-methods:
          - POST
        description: 注册接口豁免认证
        pattern-type: EXACT
        enabled: true
      - path: /health/**
        http-methods:
          - GET
        description: 健康检查接口豁免认证
        pattern-type: ANT
        enabled: true
      - path: /v3/api-docs/**
        http-methods:
          - GET
        description: Swagger文档接口豁免认证
        pattern-type: ANT
        enabled: true
```

### 3.5 日志配置

```yaml
# 日志配置
logging:
  level:
    root: info
    com.exclusive.blank: debug
  file:
    name: logs/application.log
    max-size: 10MB
    max-history: 7
```

## 4. 环境变量配置

### 4.1 生产环境变量

在生产环境中，建议使用环境变量来配置敏感信息：

| 环境变量 | 描述 | 示例 |
|----------|------|------|
| DB_PASSWORD | 数据库密码 | your_password |
| JWT_SECRET | JWT密钥 | your_secret_key |
| SSL_PASSWORD | SSL证书密码 | your_ssl_password |
| SPRING_PROFILES_ACTIVE | 激活的配置文件 | prod |

### 4.2 配置方式

1. **系统环境变量**：
   ```bash
   export DB_PASSWORD=your_password
   export JWT_SECRET=your_secret_key
   ```

2. **Docker环境变量**：
   ```bash
   docker run -e DB_PASSWORD=your_password -e JWT_SECRET=your_secret_key your-image
   ```

3. **Kubernetes环境变量**：
   ```yaml
   env:
     - name: DB_PASSWORD
       valueFrom:
         secretKeyRef:
           name: db-secret
           key: password
   ```

```properties
server.ssl.key-store-password=${SSL_PASSWORD}
server.ssl.key-store-type=PKCS12
server.ssl.key-alias=smart_registration

# 日志配置
logging.level.com.exclusive.blank=warn
```

## 6. 安全配置

### 6.1 SSL配置

1. **生成SSL证书**：
   ```bash
   keytool -genkey -alias smart_registration -keyalg RSA -keystore keystore.p12 -storetype PKCS12 -validity 365
   ```

2. **配置SSL**：
   ```yaml
   server:
     ssl:
       enabled: true
       key-store: classpath:keystore.p12
       key-store-password: your_ssl_password
       key-store-type: PKCS12
       key-alias: smart_registration
   ```

### 6.2 防火墙配置

1. **开放端口**：
   ```bash
   # CentOS
   firewall-cmd --zone=public --add-port=8080/tcp --permanent
   firewall-cmd --zone=public --add-port=80/tcp --permanent
   firewall-cmd --zone=public --add-port=443/tcp --permanent
   firewall-cmd --reload
   
   # Ubuntu
   ufw allow 8080/tcp
   ufw allow 80/tcp
   ufw allow 443/tcp
   ufw enable
   ```

### 6.3 数据库安全

1. **创建只读用户**：
   ```sql
   CREATE USER 'readonly'@'localhost' IDENTIFIED BY 'password';
   GRANT SELECT ON smart_registration_prod.* TO 'readonly'@'localhost';
   ```

2. **限制数据库访问**：
   - 只允许应用服务器访问数据库
   - 禁用远程root登录

## 8. 监控配置

### 8.1 健康检查配置

系统提供以下健康检查端点：

- **/health**：系统健康状态
- **/health/info**：系统信息
- **/health/ping**：简单心跳检查

### 8.2 Prometheus配置

1. **添加依赖**：
   ```xml
   <dependency>
       <groupId>org.springframework.boot</groupId>
       <artifactId>spring-boot-starter-actuator</artifactId>
   </dependency>
   <dependency>
       <groupId>io.micrometer</groupId>
       <artifactId>micrometer-registry-prometheus</artifactId>
   </dependency>
   ```

2. **配置Actuator**：
   ```yaml
   management:
     endpoints:
       web:
         exposure:
           include: health,info,prometheus,metrics
     endpoint:
       health:
         show-details: always
   ```

### 8.3 日志管理

1. **ELK Stack配置**：
   - 配置Logstash收集日志
   - 配置Elasticsearch存储日志
   - 配置Kibana查看日志

2. **日志轮转**：
   ```yaml
   logging:
     file:
       name: logs/application.log
       max-size: 10MB
       max-history: 7
   ```

- **使用配置中心**：如Spring Cloud Config

- **敏感信息加密**：使用Spring Cloud Config Server的加密功能
- **配置验证**：启动时验证配置的有效性
- **配置文档**：维护配置项的文档

## 10. 常见配置问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|----------|
| 数据库连接失败 | 配置错误 | 检查数据库连接字符串 |
| JWT验证失败 | 密钥错误 | 检查JWT密钥配置 |
| 端口被占用 | 端口冲突 | 修改服务器端口 |
| 内存不足 | JVM配置不当 | 调整JVM参数 |
| 性能问题 | 数据库配置不当 | 优化数据库配置 |
