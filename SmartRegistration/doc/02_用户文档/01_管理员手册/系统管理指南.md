# 系统管理指南

## 1. 概述

本文档是智能挂号预约系统的管理员操作指南，旨在帮助系统管理员了解和使用系统的管理功能。

## 2. 登录系统

### 2.1 访问地址

- **系统地址**：http://localhost:8080/api/v1
- **登录接口**：`POST /auth/login`

### 2.2 登录凭证

- **默认账号**：admin
- **默认密码**：password

### 2.3 登录流程

1. 发送登录请求：
   ```json
   {
     "username": "admin",
     "password": "password"
   }
   ```

2. 系统返回JWT令牌和用户信息：
   ```json
   {
     "code": 200,
     "message": "成功",
     "data": {
       "token": "eyJhbGciOiJIUzUxMiJ9...",
       "user": {
         "id": 1,
         "username": "admin",
         "name": "管理员",
         "role": "ADMIN",
         "status": "ACTIVE"
       }
     },
     "error": null
   }
   ```

3. 在后续请求的请求头中添加：
   ```
   Authorization: Bearer {token}
   ```

## 3. 系统管理功能

### 3.1 用户管理

#### 3.1.1 获取用户列表

- **接口**：`GET /users`
- **参数**：
  - `page`：页码（默认1）
  - `size`：每页数量（默认10）
  - `role`：角色筛选（可选）
- **响应**：用户列表和分页信息

#### 3.1.2 创建用户

- **接口**：`POST /users`
- **请求体**：
  ```json
  {
    "username": "doctor1",
    "password": "password",
    "name": "张医生",
    "gender": "男",
    "age": 35,
    "phone": "13800138000",
    "email": "doctor1@example.com",
    "role": "DOCTOR",
    "status": "ACTIVE"
  }
  ```

#### 3.1.3 更新用户

- **接口**：`PUT /users/{id}`
- **请求体**：
  ```json
  {
    "name": "李医生",
    "phone": "13900139000",
    "email": "doctor1@example.com",
    "status": "ACTIVE"
  }
  ```

#### 3.1.4 删除用户

- **接口**：`DELETE /users/{id}`

### 3.2 科室管理

#### 3.2.1 获取科室列表

- **接口**：`GET /departments`
- **响应**：科室列表

#### 3.2.2 创建科室

- **接口**：`POST /departments`
- **请求体**：
  ```json
  {
    "name": "内科",
    "description": "内科科室",
    "status": "ACTIVE"
  }
  ```

#### 3.2.3 更新科室

- **接口**：`PUT /departments/{id}`
- **请求体**：
  ```json
  {
    "name": "内科（普通）",
    "description": "普通内科科室",
    "status": "ACTIVE"
  }
  ```

#### 3.2.4 删除科室

- **接口**：`DELETE /departments/{id}`

### 3.3 医生管理

#### 3.3.1 获取医生列表

- **接口**：`GET /doctors`
- **参数**：
  - `departmentId`：科室ID（可选）
- **响应**：医生列表

#### 3.3.2 创建医生

- **接口**：`POST /doctors`
- **请求体**：
  ```json
  {
    "userId": 2,
    "departmentId": 1,
    "title": "主任医师",
    "specialty": "心血管疾病",
    "bio": "从事心血管疾病诊疗20年",
    "status": "ACTIVE"
  }
  ```

#### 3.3.3 更新医生

- **接口**：`PUT /doctors/{id}`
- **请求体**：
  ```json
  {
    "title": "副主任医师",
    "specialty": "心血管疾病、高血压",
    "status": "ACTIVE"
  }
  ```

#### 3.3.4 删除医生

- **接口**：`DELETE /doctors/{id}`

### 3.4 排班管理

#### 3.4.1 获取排班列表

- **接口**：`GET /schedules`
- **参数**：
  - `doctorId`：医生ID（可选）
  - `departmentId`：科室ID（可选）
  - `date`：日期（可选，格式：yyyy-MM-dd）
- **响应**：排班列表

#### 3.4.2 创建排班

- **接口**：`POST /schedules`
- **请求体**：
  ```json
  {
    "doctorId": 1,
    "departmentId": 1,
    "date": "2026-05-01",
    "startTime": "08:00",
    "endTime": "12:00",
    "totalSlots": 20,
    "availableSlots": 20,
    "status": "ACTIVE"
  }
  ```

#### 3.4.3 更新排班

- **接口**：`PUT /schedules/{id}`
- **请求体**：
  ```json
  {
    "totalSlots": 25,
    "availableSlots": 25,
    "status": "ACTIVE"
  }
  ```

#### 3.4.4 删除排班

- **接口**：`DELETE /schedules/{id}`

### 3.5 预约管理

#### 3.5.1 获取预约列表

- **接口**：`GET /appointments`
- **参数**：
  - `status`：状态（可选）
  - `startDate`：开始日期（可选）
  - `endDate`：结束日期（可选）
- **响应**：预约列表

#### 3.5.2 更新预约状态

- **接口**：`PUT /appointments/{id}`
- **请求体**：
  ```json
  {
    "status": "COMPLETED"
  }
  ```

#### 3.5.3 取消预约

- **接口**：`PUT /appointments/{id}/cancel`

  ```json
    "enabled": true

  ```

#### 4.1.3 删除豁免规则

  ```json

    "path": "/public/info",

  }
  ```

## 6. 常见问题

### 6.1 登录问题

- **问题**：登录失败
  **解决**：检查用户名和密码是否正确，确保用户状态为ACTIVE

- **问题**：令牌过期
  **解决**：使用刷新令牌接口获取新令牌

### 6.2 权限问题

- **问题**：无权限访问
  **解决**：检查用户角色是否有相应权限

### 6.3 数据问题

- **问题**：数据重复
  **解决**：检查唯一性约束，避免重复创建

- **问题**：数据不存在
  **解决**：检查ID是否正确，确保数据已创建

## 7. 操作流程

### 7.1 新增医生流程

1. 创建用户（角色为DOCTOR）
2. 创建科室（如果不存在）
3. 创建医生信息，关联用户和科室
4. 为医生创建排班

### 7.2 处理预约流程

1. 查看预约列表
2. 根据需要更新预约状态
3. 处理完成后标记为已完成
