# 智慧医院挂号预约系统 - 重新开发规划文档

## 一、API 接口分析与梳理

### 1.1 接口总览

| 模块 | 接口数量 | 主要功能 |
|------|----------|----------|
| 认证接口 | 3 | 登录、刷新令牌、注册 |
| 用户管理 | 5 | CRUD 操作 |
| 科室管理 | 5 | CRUD 操作 |
| 医生管理 | 7 | CRUD、按科室/状态查询 |
| 排班管理 | 9 | CRUD、按医生/日期/状态查询 |
| 预约管理 | 11 | CRUD、多维度查询 |

### 1.2 关键接口映射关系

- 用户（User）→ 医生（Doctor） → 科室（Department）
- 医生（Doctor）→ 排班（Schedule）
- 患者（User/PATIENT）→ 预约（Appointment）→ 医生（Doctor）/ 排班（Schedule）

### 1.3 数据状态枚举

| 实体 | 状态枚举 |
|------|----------|
| User/Department/Doctor | ACTIVE/INACTIVE |
| Schedule | ACTIVE/CANCELLED |
| Appointment | PENDING/CONFIRMED/COMPLETED/CANCELLED |

## 二、功能模块规划

### 2.1 核心功能模块

#### 模块 1：登录认证
- 用户登录
- Token 管理
- 角色权限控制

#### 模块 2：首页仪表盘
- 统计卡片（今日预约、科室数量、医生数量、用户数量）
- 最近预约列表
- 快捷操作入口

#### 模块 3：用户管理
- 用户列表（支持角色筛选）
- 用户新增/编辑/删除
- 用户状态管理

#### 模块 4：科室管理
- 科室列表
- 科室新增/编辑/删除
- 科室状态管理

#### 模块 5：医生管理
- 医生列表（按科室/状态筛选）
- 医生新增/编辑/删除
- 医生信息管理（关联用户和科室）

#### 模块 6：排班管理
- 排班列表（按医生/日期/状态筛选）
- 排班新增/编辑/删除/取消
- 剩余号源管理

#### 模块 7：预约管理
- 预约列表（多维度筛选）
- 预约状态变更（确认/完成/取消）
- 预约详情查看

#### 模块 8：统计报表
- 预约状态分布
- 医生工作量统计
- 科室就诊统计

## 三、技术架构设计

### 3.1 技术栈
- **前端框架**: React 19
- **路由管理**: React Router 6
- **HTTP 客户端**: Fetch API (原生)
- **组件库**: 自定义组件 (基于设计规范)
- **状态管理**: React Context + Custom Hooks
- **构建工具**: Vite

### 3.2 项目架构

```
src/
├── components/
│   ├── common/            # 通用组件 (Button, Input, Select, Modal, Table, etc.)
│   └── Layout/            # 布局组件
├── hooks/                 # 自定义 Hooks
│   ├── useAuth.js         # 认证 Hook
│   ├── useFetch.js        # 数据请求 Hook
│   ├── usePagination.js   # 分页 Hook
│   ├── useTable.js        # 表格 Hook
│   └── useDebounce.js     # 防抖 Hook
├── context/               # Context
│   ├── AuthContext.jsx
│   └── AppContext.jsx
├── pages/                 # 页面组件
│   ├── Login/
│   ├── Dashboard/
│   ├── Users/
│   ├── Departments/
│   ├── Doctors/
│   ├── Schedules/
│   ├── Appointments/
│   └── Statistics/
├── services/              # API 服务
│   ├── api.js             # API 基础配置
│   ├── authService.js
│   ├── userService.js
│   ├── departmentService.js
│   ├── doctorService.js
│   ├── scheduleService.js
│   └── appointmentService.js
├── styles/                # 全局样式
├── utils/                 # 工具函数
└── App.jsx
```

### 3.3 API 服务设计

严格遵循 API 文档，每个服务模块对应一个实体模块：
- authService: 认证相关接口
- userService: 用户管理相关接口
- departmentService: 科室管理相关接口
- doctorService: 医生管理相关接口
- scheduleService: 排班管理相关接口
- appointmentService: 预约管理相关接口

## 六、开发原则

1. **严格遵循 API 文档**: 不使用任何未在 API 文档中定义的接口
2. **模块化设计**: 高内聚、低耦合
3. **可维护性**: 代码易读、易扩展
4. **用户体验**: 清晰、直观、友好
5. **性能优先**: 代码分割、缓存、懒加载
