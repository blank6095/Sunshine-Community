# 智能挂号预约系统 数据库设计

## 1. 数据库概述

智能挂号预约系统使用 MySQL 8.0+ 作为数据库，采用关系型数据库设计，通过表之间的关联关系实现业务逻辑。数据库设计遵循第三范式，确保数据的一致性和完整性。

## 2. 核心表结构

### 2.1 用户表 (users)

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|---------|------|------|
| id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 用户ID |
| username | VARCHAR(50) | UNIQUE, NOT NULL | 用户名 |
| password | VARCHAR(100) | NOT NULL | 密码（哈希存储） |
| name | VARCHAR(50) | NOT NULL | 姓名 |
| gender | VARCHAR(10) | NOT NULL | 性别 |
| age | INT | NOT NULL | 年龄 |
| phone | VARCHAR(20) | UNIQUE, NOT NULL | 手机号码 |
| email | VARCHAR(100) | UNIQUE | 邮箱 |
| id_card | VARCHAR(18) | UNIQUE | 身份证号 |
| role | VARCHAR(20) | NOT NULL | 角色（ADMIN/DOCTOR/PATIENT） |
| status | VARCHAR(20) | NOT NULL | 状态（ACTIVE/INACTIVE） |
| created_at | TIMESTAMP | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

### 2.2 科室表 (departments)

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|---------|------|------|
| id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 科室ID |
| name | VARCHAR(50) | UNIQUE, NOT NULL | 科室名称 |
| description | VARCHAR(255) | | 科室描述 |
| status | VARCHAR(20) | NOT NULL | 状态（ACTIVE/INACTIVE） |
| created_at | TIMESTAMP | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |

### 2.3 医生表 (doctors)

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|---------|------|------|
| id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 医生ID |
| user_id | BIGINT | FOREIGN KEY (users.id), UNIQUE | 用户ID（关联用户表） |
| department_id | BIGINT | FOREIGN KEY (departments.id) | 科室ID（关联科室表） |
| title | VARCHAR(50) | NOT NULL | 职称 |
| specialty | VARCHAR(100) | | 专长 |
| bio | VARCHAR(255) | | 简介 |
| status | VARCHAR(20) | NOT NULL | 状态（ACTIVE/INACTIVE） |
| created_at | TIMESTAMP | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

### 2.4 排班表 (schedules)

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|---------|------|------|
| id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 排班ID |
| doctor_id | BIGINT | FOREIGN KEY (doctors.id) | 医生ID（关联医生表） |
| date | DATE | NOT NULL | 排班日期 |
| start_time | TIME | NOT NULL | 开始时间 |
| end_time | TIME | NOT NULL | 结束时间 |
| max_patients | INT | NOT NULL | 最大预约人数 |
| available_slots | INT | NOT NULL | 可用预约数 |
| status | VARCHAR(20) | NOT NULL | 状态（ACTIVE/CANCELLED） |
| created_at | TIMESTAMP | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

### 2.5 预约表 (appointments)

| 字段名 | 数据类型 | 约束 | 描述 |
|--------|---------|------|------|
| id | BIGINT | PRIMARY KEY, AUTO_INCREMENT | 预约ID |
| patient_id | BIGINT | FOREIGN KEY (users.id) | 患者ID（关联用户表） |
| doctor_id | BIGINT | FOREIGN KEY (doctors.id) | 医生ID（关联医生表） |
| schedule_id | BIGINT | FOREIGN KEY (schedules.id) | 排班ID（关联排班表） |

| created_at | TIMESTAMP | NOT NULL, DEFAULT CURRENT_TIMESTAMP | 创建时间 |
| updated_at | TIMESTAMP | NOT NULL, DEFAULT CURRENT_TIMESTAMP ON UPDATE CURRENT_TIMESTAMP | 更新时间 |

## 5. 数据备份与恢复方案

### 5.1 备份策略

- **每日全量备份**：
  - 时间：每天凌晨 2:00
  - 方式：使用 `mysqldump` 命令导出整个数据库
  - 存储：本地存储 + 云存储

- **每周增量备份**：
  - 时间：每周日凌晨 2:00
  - 方式：使用 MySQL 增量备份功能
  - 存储：本地存储 + 云存储

- **每月归档备份**：
  - 时间：每月最后一天凌晨 2:00
  - 方式：使用 `mysqldump` 命令导出整个数据库并压缩
  - 存储：云存储（长期保存）

### 5.2 备份存储

- **本地存储**：
  - 路径：`/backup/mysql/`
  - 保留时间：7 天
  - 自动清理过期备份

- **云存储**：
  - 服务：AWS S3 / 阿里云 OSS
  - 保留时间：
    - 每日备份：30 天
    - 每周备份：90 天
    - 每月备份：1 年
  - 加密存储

- **异地备份**：
  - 存储在不同地理位置的云存储
  - 确保数据安全性和可用性

### 5.3 恢复策略

- **恢复测试**：
  - 每月进行一次恢复测试
  - 验证备份数据的完整性和可用性
  - 记录恢复时间和过程

- **恢复流程**：
  1. 停止应用服务
  2. 备份当前数据库（如果需要）
  3. 恢复备份数据
  4. 验证数据完整性
  5. 启动应用服务
  6. 测试系统功能

- **灾难恢复**：
  - 制定详细的灾难恢复计划
  - 明确恢复步骤和责任
  - 定期演练灾难恢复流程

### 5.4 监控与告警

- **备份监控**：
  - 监控备份任务执行状态
  - 备份失败时发送告警
  - 备份存储容量监控

- **数据一致性检查**：
  - 定期检查数据库完整性
  - 检测数据异常
  - 及时发现并处理数据问题

- **告警机制**：
  - 邮件告警
  - 短信告警
  - 监控平台告警

## 6. 数据库优化

### 6.1 性能优化

- **查询优化**：
  - 合理使用索引
  - 避免全表扫描
  - 优化复杂查询

- **配置优化**：
  - 调整 MySQL 配置参数
  - 优化内存使用
  - 调整连接池大小

- **分区表**：
  - 对大型表（如预约表）使用分区
  - 按日期或其他字段分区
  - 提高查询效率

### 6.2 安全优化

- **访问控制**：
  - 最小权限原则
  - 定期更新密码
  - 限制数据库访问 IP

- **数据加密**：
  - 敏感数据加密存储
  - 传输过程加密
  - 加密密钥管理

- **审计日志**：
  - 开启 MySQL 审计日志
  - 记录关键操作
  - 定期检查审计日志
