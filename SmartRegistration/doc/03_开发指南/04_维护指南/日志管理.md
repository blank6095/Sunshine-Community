# 日志管理

## 1. 概述

本文档详细说明智能挂号预约系统的日志管理策略，帮助运维人员了解如何收集、分析和管理系统日志。

## 2. 日志体系

### 2.1 日志类型

| 日志类型 | 描述 | 存储位置 |
|---------|------|----------|
| **应用日志** | 应用运行日志 | logs/application.log |
| **访问日志** | HTTP请求日志 | logs/access.log |
| **错误日志** | 错误信息日志 | logs/error.log |
| **系统日志** | 操作系统日志 | /var/log/ |
| **数据库日志** | MySQL日志 | /var/lib/mysql/ |

### 2.2 日志级别

| 级别 | 描述 | 示例 |
|------|------|------|
| **DEBUG** | 调试信息 | 详细的调试信息 |
| **INFO** | 一般信息 | 正常的运行信息 |
| **WARN** | 警告信息 | 潜在的问题 |
| **ERROR** | 错误信息 | 错误事件 |
| **FATAL** | 致命错误 | 系统崩溃 |

## 3. 日志配置

### 3.1 应用日志配置

**application.yml**：

```yaml
logging:
  level:
    root: info
    com.exclusive.blank: debug
  file:
    name: logs/application.log
    max-size: 10MB
    max-history: 7
  pattern:
    console: "%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n"
    file: "%d{yyyy-MM-dd HH:mm:ss.SSS} [%thread] %-5level %logger{36} - %msg%n"
```

### 3.2 访问日志配置

**Nginx配置**：

```nginx
server {
    # ...
    access_log /var/log/nginx/smart-registration-access.log;
    error_log /var/log/nginx/smart-registration-error.log;
    # ...
}
```

### 3.3 数据库日志配置

**MySQL配置**：

```ini
# /etc/my.cnf
[mysqld]
log_error = /var/log/mysql/error.log
general_log = 1
general_log_file = /var/log/mysql/general.log
slow_query_log = 1
slow_query_log_file = /var/log/mysql/slow.log
long_query_time = 2
```

## 4. 日志收集

### 4.1 集中式日志收集

1. **ELK Stack**：
   - **Elasticsearch**：存储和索引日志
   - **Logstash**：收集和处理日志
   - **Kibana**：可视化和查询日志

2. **Graylog**：
   - 集中式日志管理平台
   - 支持日志收集、索引和分析

3. **Splunk**：
   - 企业级日志管理平台
   - 支持实时监控和分析

### 4.2 日志收集配置

**Logstash配置**：

```conf
input {
  file {
    path => "/opt/smart-registration/logs/*.log"
    start_position => "beginning"
    sincedb_path => "/var/lib/logstash/sincedb"
  }
}

filter {
  grok {
    match => {
      "message" => "%{TIMESTAMP_ISO8601:timestamp} \[%{DATA:thread}\] %{LOGLEVEL:level} %{DATA:logger} - %{GREEDYDATA:message}"
    }
  }
  date {
    match => ["timestamp", "yyyy-MM-dd HH:mm:ss.SSS"]
    target => "@timestamp"
  }
}

output {
  elasticsearch {
    hosts => ["localhost:9200"]
    index => "smart-registration-%{+YYYY.MM.dd}"
  }
}
```

## 5. 日志分析

### 5.1 日志查询

1. **Kibana查询**：
   - 使用KQL（Kibana Query Language）查询
   - 支持时间范围、字段过滤等

2. **Graylog查询**：
   - 使用GELF格式
   - 支持复杂的查询条件

3. **命令行工具**：
   - **grep**：搜索日志
   - **tail**：实时查看日志
   - **less**：分页查看日志

### 5.2 日志分析工具

- **ELK Stack**：完整的日志分析平台
- **Graylog**：集中式日志管理
- **Splunk**：企业级日志分析
- **Loki**：轻量级日志聚合系统

## 6. 日志监控

### 6.1 实时监控

1. **ELK Watcher**：
   - 监控日志中的特定模式
   - 触发告警

2. **Graylog Alerts**：
   - 基于条件的告警
   - 支持多种通知方式

3. **Prometheus Alertmanager**：
   - 基于指标的告警
   - 与日志系统集成

### 6.2 告警配置

**常见告警规则**：

- **错误率**：错误日志占比超过阈值
- **响应时间**：请求响应时间超过阈值
- **系统错误**：出现致命错误
- **安全事件**：可疑的安全事件

## 7. 日志保留

### 7.1 保留策略

| 日志类型 | 保留期限 | 存储方式 |
|---------|---------|----------|
| **应用日志** | 30天 | 本地文件 |
| **访问日志** | 15天 | 本地文件 |
| **错误日志** | 90天 | 本地文件 + 远程存储 |
| **系统日志** | 7天 | 本地文件 |
| **数据库日志** | 30天 | 本地文件 |

### 7.2 归档策略

1. **日志轮转**：
   - 使用logrotate工具
   - 按大小或时间轮转

2. **压缩存储**：
   - 轮转后压缩
   - 减少存储空间

3. **远程备份**：
   - 备份到云存储
   - 确保数据安全

## 8. 日志安全

### 8.1 敏感信息保护

- **日志脱敏**：
  - 对敏感信息进行脱敏处理
  - 如密码、身份证号等

- **访问控制**：
  - 限制日志访问权限
  - 只允许授权人员访问

- **传输加密**：
  - 日志传输过程加密
  - 防止数据泄露

### 8.2 日志审计

- **审计日志**：
  - 记录日志访问和修改
  - 确保日志完整性

- **合规性**：
  - 符合数据保护法规
  - 如GDPR、HIPAA等

## 9. 日志最佳实践

- **结构化日志**：使用JSON格式
- **统一格式**：统一日志格式
- **适当级别**：使用适当的日志级别
- **上下文信息**：包含足够的上下文信息
- **性能考虑**：避免过度日志

## 10. 常见日志问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|----------|
| 日志文件过大 | 日志级别过低 | 调整日志级别 |
| 日志丢失 | 磁盘空间不足 | 增加磁盘空间 |
| 日志分析困难 | 格式不统一 | 统一日志格式 |
| 性能下降 | 日志过多 | 优化日志输出 |
