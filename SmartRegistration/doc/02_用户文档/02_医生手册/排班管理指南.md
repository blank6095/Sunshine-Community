# 排班管理指南

## 1. 概述

本文档详细说明智能挂号预约系统的排班管理功能，帮助医生和管理员了解如何创建、管理和优化排班。

## 2. 排班管理基础

### 2.1 排班概念

- **排班**：医生在特定日期和时间段的工作安排
- **号源**：排班中可预约的名额
- **时间段**：排班的开始和结束时间
- **状态**：排班的状态（ACTIVE、INACTIVE、CANCELLED）

### 2.2 排班规则

- 每个排班对应一个医生
- 每个排班对应一个科室
- 排班时间不能重叠
- 号源数量应合理设置

## 3. 排班管理功能

### 3.1 创建排班

#### 3.1.1 接口信息

- **接口**：`POST /schedules`
- **权限**：医生（只能创建自己的排班）、管理员
- **请求体**：
  ```json
  {
    "doctorId": 1,
    "departmentId": 1,
    "date": "2026-05-01",
    "startTime": "08:00",
    "endTime": "12:00",
    "totalSlots": 20,
    "availableSlots": 20,
    "status": "ACTIVE"
  }
  ```

#### 3.1.2 参数说明

| 参数 | 类型 | 描述 | 示例 |
|------|------|------|------|
| doctorId | Long | 医生ID | 1 |
| departmentId | Long | 科室ID | 1 |
| date | String | 排班日期（yyyy-MM-dd） | "2026-05-01" |
| startTime | String | 开始时间（HH:MM） | "08:00" |
| endTime | String | 结束时间（HH:MM） | "12:00" |
| totalSlots | Integer | 总号源数 | 20 |
| availableSlots | Integer | 可用号源数 | 20 |
| status | String | 状态（ACTIVE/INACTIVE） | "ACTIVE" |

### 3.2 更新排班

#### 3.2.1 接口信息

- **接口**：`PUT /schedules/{id}`
- **权限**：医生（只能更新自己的排班）、管理员
- **请求体**：
  ```json
  {
    "startTime": "08:30",
    "endTime": "12:30",
    "totalSlots": 25,
    "availableSlots": 25,
    "status": "ACTIVE"
  }
  ```

### 3.3 删除排班

#### 3.3.1 接口信息

- **接口**：`DELETE /schedules/{id}`
- **权限**：医生（只能删除自己的排班）、管理员

### 3.4 查询排班

#### 3.4.1 获取我的排班

- **接口**：`GET /schedules/doctor/me`
- **权限**：医生
- **参数**：
  - `date`：日期（可选，格式：yyyy-MM-dd）
  - `status`：状态（可选）
- **响应**：我的排班列表

#### 3.4.2 获取科室排班

- **接口**：`GET /schedules/department/{departmentId}`
- **权限**：所有角色
- **参数**：
  - `date`：日期（可选，格式：yyyy-MM-dd）
- **响应**：科室排班列表

#### 3.4.3 获取医生排班

- **接口**：`GET /schedules/doctor/{doctorId}`
- **权限**：所有角色
- **参数**：
  - `date`：日期（可选，格式：yyyy-MM-dd）
- **响应**：医生排班列表

## 4. 排班管理流程

### 4.1 周排班计划

1. **制定排班计划**：根据医生 availability 制定每周排班计划
2. **创建排班**：在系统中创建排班记录
3. **发布排班**：设置排班状态为ACTIVE
4. **调整排班**：根据实际情况调整排班

### 4.2 排班调整流程

1. **评估需要**：根据患者需求和医生 availability 评估调整需要
2. **通知患者**：如果调整影响现有预约，提前通知患者
3. **更新排班**：在系统中更新排班信息
4. **确认调整**：确保调整后的排班合理可行

### 4.3 紧急情况处理

1. **识别紧急情况**：如医生临时有事需要取消排班
2. **取消排班**：将排班状态设置为CANCELLED
3. **通知患者**：通过系统通知或电话通知受影响的患者
4. **安排替代**：如有可能，安排其他医生替代

## 5. 排班优化策略

### 5.1 号源设置

- **根据医生经验**：经验丰富的医生可设置更多号源
- **根据科室需求**：热门科室可适当增加号源
- **根据时间段**：高峰期可设置更多号源

### 5.2 时间安排

- **合理分配时间**：每个患者预留足够的就诊时间
- **避免连续排班**：给医生适当的休息时间
- **考虑患者方便**：设置在患者方便的时间段

### 5.3 排班监控

- **监控预约率**：根据预约率调整号源
- **监控就诊时间**：根据实际就诊时间调整排班
- **监控患者满意度**：根据患者反馈优化排班

## 6. 常见问题

### 6.1 排班冲突

- **问题**：创建排班时提示时间冲突
  **解决**：检查该时间段是否已有排班，选择其他时间段

### 6.2 号源不足

- **问题**：患者反映号源不足
  **解决**：根据需求增加号源，或调整排班时间

### 6.3 排班取消

- **问题**：需要取消已发布的排班
  **解决**：将排班状态设置为CANCELLED，并通知受影响的患者

### 6.4 预约溢出

- **问题**：预约数量超过号源
  **解决**：检查availableSlots是否正确更新，及时调整

## 7. 排班统计

### 7.1 排班利用率

- **计算公式**：(总预约数 / 总号源数) × 100%
- **目标**：70%-90%

### 7.2 科室排班统计

- 各科室的排班数量
- 各科室的预约率
- 各科室的患者满意度

### 7.3 医生排班统计

- 医生的排班时间
- 医生的预约数量
- 医生的平均就诊时间
