# 医院挂号系统数据规划文档

## 1. 系统概述

医院挂号系统是基于若依框架开发的医院专属挂号系统，旨在为患者提供便捷的在线挂号服务，同时为医院管理提供高效的预约管理功能。系统主要包括患者管理、医生管理、科室管理、号源管理、预约挂号和权限管理等核心模块。

### 2.2 数据库表关系

- 科室信息表 (department_info) 与 医生信息表 (doctor_info)：一对多关系，一个科室有多个医生
- 医生信息表 (doctor_info) 与 挂号时间表 (registration_schedule)：一对多关系，一个医生有多个排班
- 挂号时间表 (registration_schedule) 与 预约记录表 (appointment_record)：一对多关系，一个排班有多个预约
- 患者信息表 (patient_info) 与 预约记录表 (appointment_record)：一对多关系，一个患者有多个预约
- 科室信息表 (department_info) 与 预约记录表 (appointment_record)：一对多关系，一个科室有多个预约

### 2.3 数据库索引设计

- 科室信息表：`uk_dept_code` (dept_code)
- 医生信息表：`uk_doctor_code` (doctor_code), `idx_dept_id` (dept_id)
- 患者信息表：`uk_id_card` (id_card)
- 挂号时间表：`idx_doctor_id` (doctor_id), `idx_dept_id` (dept_id), `idx_schedule_date` (schedule_date)
- 预约记录表：`uk_registration_no` (registration_no), `idx_patient_id` (patient_id), `idx_doctor_id` (doctor_id), `idx_dept_id` (dept_id), `idx_schedule_id` (schedule_id)

## 3. 业务流程设计

### 3.1 患者注册与登录流程

1. 患者访问系统，点击注册按钮
2. 填写个人信息（姓名、身份证号、手机号等）
3. 系统验证信息合法性，创建患者账户
4. 患者使用身份证号和密码登录系统
5. 系统验证登录信息，生成登录凭证

### 3.2 科室选择与医生查询流程

1. 患者登录系统后，进入科室选择页面
2. 系统展示所有可用科室列表
3. 患者选择科室后，系统展示该科室的医生列表
4. 患者可以查看医生详情（职称、专业特长等）

### 3.3 号源查询与在线挂号流程

1. 患者选择医生后，进入号源查询页面
2. 系统展示该医生的可挂号日期和时段
3. 患者选择预约日期和时段
4. 系统检查该时段是否有可用号源
5. 患者确认预约信息，提交挂号请求
6. 系统生成挂号编号，更新号源状态

### 3.4 预约管理与挂号记录查询流程

1. 患者登录系统后，进入预约管理页面
2. 系统展示患者的所有预约记录
3. 患者可以查看预约详情、取消预约等操作
4. 系统更新预约状态

### 3.5 医生排班管理流程

1. 管理员登录系统，进入排班管理页面
2. 选择医生和日期，设置可挂号数量
3. 系统保存排班信息
4. 系统根据排班信息生成号源

## 4. 权限控制设计

### 4.1 角色设计

- **患者角色**：可以注册、登录、查询医生信息、预约挂号、管理个人预约记录
- **医生角色**：可以登录、查看自己的排班信息、查看自己的预约记录、处理患者就诊
- **管理员角色**：可以管理科室、医生、患者信息、排班信息、预约记录等所有系统功能

### 4.2 权限设计

| 功能模块 | 患者角色 | 医生角色 | 管理员角色 |
| :--- | :--- | :--- | :--- |
| 患者注册与登录 | ✅ | ❌ | ❌ |
| 科室查询 | ✅ | ✅ | ✅ |
| 医生查询 | ✅ | ✅ | ✅ |
| 号源查询 | ✅ | ✅ | ✅ |
| 在线挂号 | ✅ | ❌ | ❌ |
| 预约管理 | ✅（个人） | ✅（个人） | ✅（全部） |

## 5. 接口设计

### 5.1 患者相关接口

- `POST /patient/register` - 患者注册
- `POST /patient/login` - 患者登录
- `GET /patient/info` - 获取患者个人信息
- `PUT /patient/info` - 更新患者个人信息

### 5.2 科室相关接口

- `GET /department/list` - 获取科室列表
- `GET /department/detail/{deptId}` - 获取科室详情

### 5.3 医生相关接口

- `GET /doctor/list` - 获取医生列表
- `GET /doctor/listByDept/{deptId}` - 按科室获取医生列表
- `GET /doctor/detail/{doctorId}` - 获取医生详情

### 5.4 号源相关接口

- `GET /schedule/list` - 获取号源列表
- `GET /schedule/listByDoctor/{doctorId}` - 按医生获取号源列表
- `GET /schedule/detail/{scheduleId}` - 获取号源详情

### 5.5 预约相关接口

- `POST /appointment/create` - 创建预约
- `GET /appointment/list` - 获取预约列表
- `GET /appointment/detail/{appointmentId}` - 获取预约详情
- `PUT /appointment/cancel/{appointmentId}` - 取消预约

## 6. 数据安全设计

### 6.1 数据传输安全

- 使用HTTPS协议进行数据传输
- 接口请求参数进行加密处理
- 敏感信息（如身份证号、手机号）进行脱敏处理

### 6.2 数据存储安全

- 数据库密码加密存储
- 敏感数据加密存储
- 定期备份数据库

### 6.3 访问控制安全

- 基于角色的权限控制
- 登录失败次数限制
- 会话超时管理

## 7. 系统性能优化

### 7.1 数据库优化

- 合理设计索引
- 优化查询语句
- 分表分库（当数据量较大时）

### 7.2 系统优化

- 缓存热点数据
- 异步处理耗时操作
- 负载均衡（当用户量较大时）
