## 2 H5平台部署

### 2.1 构建H5应用

```bash
# 执行生产构建
pnpm build:h5
```

构建完成后，产物目录为 `dist/build/h5`。

### 2.2 Nginx配置

创建Nginx配置文件：

```nginx
server {
    listen 80;
    server_name your-domain.com;

    root /var/www/hospital/dist;
    index index.html;

    # Gzip压缩
    gzip on;
    gzip_types text/plain text/css application/json application/javascript text/xml application/xml;
    gzip_min_length 1024;

    # 静态资源缓存
    location ~* \.(js|css|png|jpg|jpeg|gif|ico|svg|woff|woff2|ttf|eot)$ {
        expires 30d;
        add_header Cache-Control "public, immutable";
    }

    # API代理
    location /api/ {
        proxy_pass http://backend-server:8080/api/;
        proxy_set_header Host $host;
        proxy_set_header X-Real-IP $remote_addr;
    }

    # Vue History模式路由
    location / {
        try_files $uri $uri/ /index.html;
    }
}
```

### 2.3 部署步骤

```bash
# 1. 上传构建产物到服务器
scp -r dist/build/h5/* user@server:/var/www/hospital/dist/

# 2. 重启Nginx
ssh user@server "sudo systemctl restart nginx"

# 3. 验证部署
curl -I https://your-domain.com
```

### 2.4 HTTPS配置

如果需要启用HTTPS，可以使用Let's Encrypt免费证书：

```bash
# 安装Certbot
sudo apt install certbot python3-certbot-nginx

# 获取证书
sudo certbot --nginx -d your-domain.com

# 自动续期测试
sudo certbot renew --dry-run
```

## 3 微信小程序部署

### 3.1 构建微信小程序

```bash
# 执行微信小程序构建
pnpm build:mp-weixin
```

构建完成后，产物目录为 `dist/build/mp-weixin`。

### 3.2 微信公众平台配置

1. 登录 [微信公众平台](https://mp.weixin.qq.com/)
2. 进入 `开发管理` -> `开发设置`
3. 配置服务器域名：
   - request合法域名：`https://api.example.com`
   - socket合法域名
   - uploadFile合法域名
   - downloadFile合法域名

### 3.3 上传代码

**方式一：使用微信开发者工具**

1. 打开微信开发者工具
2. 导入项目 `dist/build/mp-weixin` 目录
3. 填写AppID
4. 点击 `上传` 按钮
5. 填写版本号和备注

**方式二：使用CI/CD**

创建GitHub Actions工作流 `.github/workflows/deploy-mp-weixin.yml`：

```yaml
name: Deploy MP-Weixin

on:
  push:
    tags:
      - 'v*'

jobs:
  deploy:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'pnpm'

      - name: Install dependencies
        run: pnpm install

      - name: Build
        run: pnpm build:mp-weixin
        env:
          VITE_API_BASE_URL: ${{ secrets.MP_API_BASE_URL }}

      - name: Upload
        uses: actions/upload-artifact@v3
        with:
          name: mp-weixin
          path: dist/build/mp-weixin
```

### 3.4 提交审核

1. 登录微信公众平台
2. 进入 `版本管理`
3. 选择要提交的版本
4. 填写版本说明
5. 点击 `提交审核`

**审核注意事项：**
- 确保所有功能符合微信小程序规范
- 检查是否有诱导分享行为
- 确保隐私政策已配置
- 准备好软件著作权证书（如需）

## 4 Android App部署

### 4.1 构建Android应用

```bash
# 执行Android构建
pnpm build:app
```

构建完成后，产物目录为 `dist/build/app-android`。

### 4.2 生成签名文件

如果还没有签名文件，生成一个：

```bash
# 生成签名文件
keytool -genkey -v -keystore my-release-key.jks -keyalg RSA -keysize 2048 -validity 10000 -alias my-key-alias
```

### 4.3 配置签名信息

在 `AndroidManifest.xml` 或使用Gradle配置签名：

```gradle
android {
    signingConfigs {
        release {
            storeFile file('my-release-key.jks')
            storePassword 'your-store-password'
            keyAlias 'my-key-alias'
            keyPassword 'your-key-password'
        }
    }
    buildTypes {
        release {
            signingConfig signingConfigs.release
            minifyEnabled true
            proguardFiles getDefaultProguardFile('proguard-android-optimize.txt'), 'proguard-rules.pro'
        }
    }
}
```

## 5 iOS App部署

### 5.1 构建iOS应用

```bash
# 执行iOS构建
pnpm build:app-ios
```

构建完成后，使用Xcode打开项目进行进一步配置。

### 5.2 Xcode配置

1. 打开 `dist/build/app-ios` 目录中的 Xcode 项目
2. 配置签名证书：
   - 选择Team
   - 选择Provisioning Profile
3. 设置版本号和构建号
4. 配置App Icons和启动图

### 5.3 App Store Connect配置

1. 登录 [App Store Connect](https://appstoreconnect.apple.com/)
2. 创建新应用
3. 填写应用信息：
   - 应用名称
   - 副标题
   - 描述
   - 关键词
   - 隐私政策URL
   - 支持URL
4. 上传构建产物

### 5.4 上传到App Store

**方式一：使用Xcode**

1. 在Xcode中选择 `Product` -> `Archive`
2. 在Organizer窗口中选择构建
3. 点击 `Distribute App`
4. 选择 `App Store Connect`
5. 按照指引完成上传

**方式二：使用Transporter**

1. 下载 [Transporter](https://apps.apple.com/us/app/transporter/id1450874784) 应用
2. 登录Apple ID
3. 上传构建的.ipa文件
4. 等待处理完成

### 5.5 提交审核

1. 在App Store Connect中选择构建版本
2. 填写测试账号（如需要）
3. 填写联系信息和备注
4. 点击 `提交以供审核`

**审核注意事项：**
- 确保应用符合App Store审核指南
- 提供有效的测试账号
- 确保隐私政策已配置
- 检查应用内购配置

## 6 CI/CD自动化部署

### 6.1 GitHub Actions配置

创建 `.github/workflows/deploy.yml`：

```yaml
name: Deploy

on:
  push:
    branches:
      - main
    tags:
      - 'v*'

jobs:
  deploy-h5:
    runs-on: ubuntu-latest
    steps:
      - uses: actions/checkout@v3

      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'pnpm'

      - name: Install & Build
        run: |
          pnpm install
          pnpm build:h5

      - name: Deploy to Server
        uses: appleboy/scp-action@master
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          key: ${{ secrets.SERVER_SSH_KEY }}
          source: "dist/build/h5/*"
          target: "/var/www/hospital/dist"

      - name: Restart Nginx
        uses: appleboy/ssh-action@master
        with:
          host: ${{ secrets.SERVER_HOST }}
          username: ${{ secrets.SERVER_USER }}
          key: ${{ secrets.SERVER_SSH_KEY }}
          script: |
            sudo systemctl restart nginx

  deploy-mp-weixin:
    runs-on: ubuntu-latest
    if: startsWith(github.ref, 'refs/tags/v')
    steps:
      - uses: actions/checkout@v3

      - name: Setup Node
        uses: actions/setup-node@v3
        with:
          node-version: '18'
          cache: 'pnpm'

      - name: Install & Build
        run: |
          pnpm install
          pnpm build:mp-weixin

      - name: Upload Artifact
        uses: actions/upload-artifact@v3
        with:
          name: mp-weixin
          path: dist/build/mp-weixin
```

### 6.2 服务器密钥配置

在GitHub仓库的 `Settings` -> `Secrets` 中配置：

- `SERVER_HOST`: 服务器地址
- `SERVER_USER`: 服务器用户名
- `SERVER_SSH_KEY`: SSH私钥

### 7.3 回滚方案

**H5回滚：**

```bash
# 查看历史版本
ls /var/www/hospital/dist/

# 回滚到指定版本
cp /var/www/hospital/dist/v1.2.0/* /var/www/hospital/dist/current/

# 重启Nginx
sudo systemctl restart nginx
```

**版本标记策略：**

```
/var/www/hospital/dist/
├── current/      # 当前版本
├── v1.0.0/       # 版本1.0.0
├── v1.1.0/       # 版本1.1.0
└── v1.2.0/       # 版本1.2.0
```
