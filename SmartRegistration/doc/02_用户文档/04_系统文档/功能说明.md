# 功能说明

## 1. 概述

本文档详细说明智能挂号预约系统的功能模块和使用方法，帮助用户了解系统的整体功能和使用流程。

## 2. 系统功能模块

### 2.1 认证模块

#### 2.1.1 登录

- **功能**：用户登录系统
- **接口**：`POST /auth/login`
- **参数**：
  - `username`：用户名
  - `password`：密码
- **响应**：JWT令牌和用户信息

#### 2.1.2 注册

- **功能**：用户注册
- **接口**：`POST /auth/register`
- **参数**：
  - `username`：用户名
  - `password`：密码
  - `name`：姓名
  - `gender`：性别
  - `age`：年龄
  - `phone`：电话
  - `email`：邮箱
  - `role`：角色
  - `status`：状态
- **响应**：注册成功的用户信息

#### 2.1.3 刷新令牌

- **功能**：刷新JWT令牌
- **接口**：`POST /auth/refresh`
- **参数**：
  - `token`：当前令牌
- **响应**：新的JWT令牌

### 2.2 用户模块

#### 2.2.1 获取用户列表

- **功能**：获取用户列表（管理员）
- **接口**：`GET /users`
- **参数**：
  - `page`：页码
  - `size`：每页数量
  - `role`：角色筛选
- **响应**：用户列表和分页信息

#### 2.2.2 获取用户详情

- **功能**：获取用户详情
- **接口**：`GET /users/{id}`
- **响应**：用户详情

#### 2.2.3 创建用户

- **功能**：创建用户（管理员）
- **接口**：`POST /users`
- **参数**：用户信息
- **响应**：创建的用户信息

#### 2.2.4 更新用户

- **功能**：更新用户信息
- **接口**：`PUT /users/{id}`
- **参数**：用户信息
- **响应**：更新后的用户信息

#### 2.2.5 删除用户

- **功能**：删除用户（管理员）
- **接口**：`DELETE /users/{id}`
- **响应**：删除成功信息

#### 2.2.6 获取当前用户信息

- **功能**：获取当前登录用户信息
- **接口**：`GET /users/me`
- **响应**：当前用户信息

### 2.5 排班模块

- **接口**：`GET /schedules`
- **参数**：

#### 2.5.4 更新排班

### 2.8 认证豁免模块

#### 2.8.1 获取豁免状态

- **功能**：获取认证豁免状态（管理员）
- **接口**：`GET /admin/auth-exemption/status`
- **响应**：豁免状态

#### 2.8.2 获取豁免规则

- **功能**：获取认证豁免规则（管理员）
- **接口**：`GET /admin/auth-exemption/rules`
- **响应**：豁免规则列表

#### 2.8.3 添加豁免规则

- **功能**：添加认证豁免规则（管理员）
- **接口**：`POST /admin/auth-exemption/rules`
- **参数**：
  - `path`：路径
  - `httpMethods`：HTTP方法列表
  - `description`：描述
  - `patternType`：匹配类型
  - `enabled`：是否启用
- **响应**：添加的规则

#### 2.8.4 删除豁免规则

- **功能**：删除认证豁免规则（管理员）
- **接口**：`DELETE /admin/auth-exemption/rules`
- **参数**：
  - `path`：路径
  - `httpMethods`：HTTP方法列表
- **响应**：删除成功信息

#### 2.8.5 重新加载配置

- **功能**：重新加载认证豁免配置（管理员）
- **接口**：`POST /admin/auth-exemption/reload`
- **响应**：重新加载成功信息

#### 2.8.6 验证配置

- **功能**：验证认证豁免配置（管理员）
- **接口**：`GET /admin/auth-exemption/validate`
- **响应**：配置验证结果

#### 2.8.7 测试路径豁免

- **功能**：测试路径是否豁免（管理员）
- **接口**：`POST /admin/auth-exemption/test`
- **参数**：
  - `path`：路径
  - `httpMethod`：HTTP方法
- **响应**：测试结果

## 3. 系统工作流程

### 3.1 患者预约流程

1. 患者注册/登录
2. 浏览科室列表
3. 选择科室查看医生
4. 查看医生排班
5. 选择合适的排班进行预约
6. 提交预约信息
7. 收到预约确认
8. 按时就诊
9. 查看预约记录

### 3.2 医生工作流程

1. 医生登录
2. 管理个人排班
3. 查看预约列表
4. 处理患者预约
5. 更新预约状态
6. 查看工作统计

### 3.3 管理员工作流程

1. 管理员登录
2. 管理用户账号
3. 管理科室和医生
4. 监控系统运行
5. 配置系统参数
6. 查看系统统计

## 4. 系统特性

### 4.1 安全性

- 基于JWT的身份认证
- 密码BCrypt加密
- 基于角色的访问控制
- 输入验证和防注入

### 4.2 可靠性

- 事务管理
- 错误处理
- 数据备份
- 系统监控

### 4.3 可扩展性

- 模块化设计
- 分层架构
- 多环境支持
- 容器化部署

### 4.4 易用性

- RESTful API设计
- 统一响应格式
- Swagger文档
- 健康检查端点

## 5. 技术特性

### 5.1 技术栈

- Spring Boot 4.0.5
- Spring Security 6.x
- Spring Data JPA
- MySQL 8.0+
- JWT 0.9.1
- Swagger 3.0

### 5.2 系统架构

- **控制器层**：处理HTTP请求和响应
- **服务层**：实现业务逻辑
- **数据访问层**：与数据库交互
- **模型层**：定义数据结构
- **安全层**：处理认证和授权
- **中间件**：实现横切关注点

## 6. 总结

智能挂号预约系统是一个功能完整、安全可靠的医疗预约管理系统，通过提供清晰的API接口和完善的功能模块，为医院、医生和患者提供了便捷的挂号预约服务。系统采用现代化的技术栈和架构设计，具有良好的可扩展性和可维护性，能够满足不同规模医院的需求。
