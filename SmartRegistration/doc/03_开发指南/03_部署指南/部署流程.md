# 部署流程

## 1. 概述

本文档详细说明智能挂号预约系统的部署流程，帮助运维人员了解如何部署和管理系统。

## 2. 部署环境

### 2.1 生产环境要求

| 环境 | 配置要求 |
|------|----------|
| **服务器** | 4核8G以上，50GB以上磁盘空间 |
| **操作系统** | CentOS 7+ 或 Ubuntu 18.04+ |
| **JDK** | OpenJDK 17+ |
| **MySQL** | 8.0+ |
| **Nginx** | 1.18+ |
| **Redis** | 6.0+ (可选，用于缓存) |

### 2.2 网络要求

- **端口**：8080（应用）、80/443（Nginx）、3306（MySQL）
- **防火墙**：开放必要的端口
- **网络带宽**：至少100Mbps

## 3. 部署前准备

### 3.1 环境准备

1. **安装JDK**：
   ```bash
   # CentOS
   yum install java-17-openjdk-devel
   
   # Ubuntu
   apt install openjdk-17-jdk
   ```

2. **安装MySQL**：
   - 参考MySQL官方文档安装
   - 配置MySQL服务

3. **安装Nginx**：
   ```bash
   # CentOS
   yum install nginx
   
   # Ubuntu
   apt install nginx
   ```

4. **安装Git**：
   ```bash
   # CentOS
   yum install git
   
   # Ubuntu
   apt install git
   ```

### 3.2 数据库准备

1. **创建数据库**：
   ```bash
   mysql -u root -p
   CREATE DATABASE smart_registration_prod;
   ```

2. **创建数据库用户**：
   ```bash
   CREATE USER 'smart_registration'@'localhost' IDENTIFIED BY 'your_password';
   GRANT ALL PRIVILEGES ON smart_registration_prod.* TO 'smart_registration'@'localhost';
   FLUSH PRIVILEGES;
   ```

3. **导入SQL脚本**：
   ```bash
   mysql -u smart_registration -p smart_registration_prod < sql/init.sql
   mysql -u smart_registration -p smart_registration_prod < sql/data.sql
   ```

## 4. 应用部署

### 4.1 代码部署

1. **克隆代码**：
   ```bash
   cd /opt
   git clone https://github.com/your-repo/smart-registration.git
   cd smart-registration
   ```

2. **构建项目**：
   ```bash
   mvn clean package -DskipTests
   ```

3. **配置应用**：
   - 编辑`src/main/resources/application-prod.properties`
   - 配置数据库连接信息和其他参数

### 4.2 服务部署

1. **创建系统服务**：
   - 创建`/etc/systemd/system/smart-registration.service`文件
   - 内容如下：
   ```ini
   [Unit]
   Description=Smart Registration System
   After=syslog.target network.target

   [Service]
   Type=simple
   User=root
   WorkingDirectory=/opt/smart-registration
   ExecStart=/usr/bin/java -jar target/blank-0.0.1-SNAPSHOT.jar --spring.profiles.active=prod
   Restart=always
   RestartSec=10

   [Install]
   WantedBy=multi-user.target
   ```

2. **启动服务**：
   ```bash
   systemctl daemon-reload
   systemctl start smart-registration
   systemctl enable smart-registration
   ```

3. **检查服务状态**：
   ```bash
   systemctl status smart-registration
   ```

### 4.3 Nginx配置

1. **配置Nginx**：
   - 创建`/etc/nginx/conf.d/smart-registration.conf`文件
   - 内容如下：
   ```nginx
   server {
       listen 80;
       server_name example.com;
       return 301 https://$host$request_uri;
   }

   server {
       listen 443 ssl;
       server_name example.com;

       ssl_certificate /etc/letsencrypt/live/example.com/fullchain.pem;
       ssl_certificate_key /etc/letsencrypt/live/example.com/privkey.pem;

       location / {
           proxy_pass http://localhost:8080;
           proxy_set_header Host $host;
           proxy_set_header X-Real-IP $remote_addr;
           proxy_set_header X-Forwarded-For $proxy_add_x_forwarded_for;
           proxy_set_header X-Forwarded-Proto $scheme;
       }
   }
   ```

2. **重启Nginx**：
   ```bash
   systemctl restart nginx
   ```

## 5. 部署验证

### 5.1 服务验证

1. **检查应用状态**：
   ```bash
   curl http://localhost:8080/api/v1/health
   ```

2. **检查Nginx状态**：
   ```bash
   curl https://example.com/api/v1/health
   ```

3. **检查数据库连接**：
   - 查看应用日志
   - 确认数据库连接正常

### 5.2 功能验证

1. **测试登录功能**：
   ```bash
   curl -X POST https://example.com/api/v1/auth/login \
     -H "Content-Type: application/json" \
     -d '{"username":"admin","password":"password"}'
   ```

2. **测试其他功能**：
   - 测试科室列表
   - 测试医生列表
   - 测试预约功能

## 6. 部署策略

### 6.1 蓝绿部署

1. **准备环境**：
   - 蓝色环境（当前版本）
   - 绿色环境（新版本）

2. **部署流程**：
   - 在绿色环境部署新版本
   - 验证绿色环境
   - 切换流量到绿色环境
   - 监控系统运行
   - 如出现问题，切回蓝色环境

### 6.2 滚动部署

1. **部署流程**：
   - 逐台服务器部署新版本
   - 每台服务器部署后验证
   - 确保系统始终可用

### 6.3 容器化部署

1. **Docker部署**：
   - 创建Dockerfile
   - 构建镜像
   - 运行容器

2. **Kubernetes部署**：
   - 创建Deployment
   - 创建Service
   - 配置Ingress

## 7. 部署管理

### 7.1 版本管理

- **版本号规范**：使用语义化版本（如1.0.0）
- **版本发布**：创建版本标签
- **版本回滚**：支持快速回滚到之前版本

### 7.2 配置管理

- **配置分离**：使用环境变量或配置中心
- **敏感信息**：使用密钥管理服务
- **配置版本控制**：配置文件纳入版本控制

### 7.3 监控管理

- **应用监控**：使用Prometheus和Grafana
- **日志管理**：使用ELK Stack
- **告警管理**：配置关键指标告警

## 8. 常见部署问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|----------|
| 服务启动失败 | 端口被占用 | 检查端口占用情况 |
| 数据库连接失败 | 配置错误 | 检查数据库配置 |
| Nginx 404 | 配置错误 | 检查Nginx配置 |
| 服务无响应 | 内存不足 | 增加服务器内存 |
| SSL证书错误 | 证书过期 | 更新SSL证书 |
