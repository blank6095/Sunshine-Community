# 测试环境搭建

## 1. 概述

本文档详细说明智能挂号预约系统的测试环境搭建步骤，帮助测试人员快速搭建测试环境。

## 2. 环境要求

### 2.1 硬件要求

- **CPU**：至少2核
- **内存**：至少4GB
- **磁盘**：至少30GB可用空间

### 2.2 软件要求

| 软件 | 版本 | 用途 |
|------|------|------|
| JDK | 17+ | Java运行环境 |
| MySQL | 8.0+ | 数据库 |
| Maven | 3.6+ | 构建工具 |
| Jenkins | 2.300+ | CI/CD工具 |
| SonarQube | 8.9+ | 代码质量分析 |
| JMeter | 5.4+ | 性能测试工具 |

### 3.1 基础环境搭建

1. **JDK安装**：
   - 参考开发环境搭建文档

2. **MySQL安装**：
   - 参考开发环境搭建文档

## 4. 测试环境配置

### 4.1 数据库配置

1. **创建测试数据库**：
   ```bash
   mysql -u root -p
   CREATE DATABASE smart_registration_test;
   ```

2. **导入测试数据**：
   ```bash
   mysql -u root -p smart_registration_test < sql/init.sql
   mysql -u root -p smart_registration_test < sql/data.sql
   ```

### 4.2 应用配置

1. **修改配置文件**：
   - 编辑`src/main/resources/application-test.properties`
   - 配置测试数据库连接信息

2. **配置环境变量**：
   - 设置测试环境的环境变量

## 5. 测试环境部署

### 5.1 自动化部署

1. **配置Jenkins任务**：
   - 创建构建任务

2. **自动化测试**：
   - 配置单元测试

### 5.2 手动部署

1. **构建项目**：
   ```bash
   mvn clean package -DskipTests
   ```

   ```bash
   java -jar -Dspring.profiles.active=test target/blank-0.0.1-SNAPSHOT.jar
   ```

### 6.2 环境监控

## 7. 测试类型

### 7.1 单元测试

- **测试范围**：单个方法或类
- **测试工具**：JUnit、Mockito
- **运行命令**：
  ```bash
  mvn test
  ```

### 7.2 集成测试

- **测试范围**：多个组件的集成
- **测试工具**：Spring Test
- **运行命令**：
  ```bash
  mvn integration-test
  ```

### 7.3 功能测试

- **测试范围**：系统功能
- **测试工具**：Selenium、Postman
- **测试方法**：API测试、UI测试

### 7.4 性能测试

- **测试范围**：系统性能
- **测试工具**：JMeter
- **测试内容**：响应时间、并发处理、吞吐量

### 7.5 安全测试

- **测试范围**：系统安全
- **测试工具**：OWASP ZAP
- **测试内容**：SQL注入、XSS、CSRF

### 8.1 测试数据生成

### 8.2 测试数据清理

## 9. 测试环境问题排查

### 9.1 常见问题

| 问题 | 可能原因 | 解决方案 |
|------|---------|----------|
| 测试失败 | 代码问题 | 检查错误日志 |
| 环境不稳定 | 资源不足 | 增加系统资源 |
| 数据不一致 | 数据污染 | 清理测试数据 |
| 网络问题 | 网络连接 | 检查网络配置 |

### 9.2 日志查看

- **应用日志**：查看应用运行日志
- **测试日志**：查看测试执行日志
- **系统日志**：查看系统运行日志

## 10. 测试环境最佳实践

- **环境一致性**：测试环境应与生产环境保持一致
- **数据隔离**：使用独立的测试数据
- **自动化测试**：尽可能自动化测试过程
- **持续集成**：集成CI/CD流程
- **性能监控**：监控测试环境性能
