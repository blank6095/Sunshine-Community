# 权限变更报告 - 患者创建预约订单权限优化

## 变更概述

**变更日期**: 2026-04-24  
**变更实施人员**: AI Assistant  
**变更类型**: 权限增强与安全加固  
**影响范围**: 患者角色创建预约订单功能  
**风险等级**: 低  

## 变更目的

确保患者账户具备创建预约订单的完整权限,同时增强安全性验证,防止患者为其他账户创建预约。

## 变更内容

### 1. 权限配置现状确认

**位置**: `src/main/java/com/exclusive/blank/controller/AppointmentController.java`

**已有权限配置**:
- **端点**: `POST /appointments`
- **权限注解**: `@PreAuthorize("hasRole('PATIENT')")`
- **功能**: 允许具有 PATIENT 角色的用户创建预约订单

**验证结果**: ✅ 患者角色已具备创建预约订单的基础权限

### 2. 安全增强 - 患者身份验证

**变更文件**:
- `AppointmentController.java`
- `AppointmentService.java`
- `ErrorCode.java`

**变更详情**:

#### 2.1 Controller 层增强
**文件**: `AppointmentController.java:40-47`

**变更前**:
```java
@PostMapping
@PreAuthorize("hasRole('PATIENT')")
public ResponseEntity<ApiResponse<AppointmentResponse>> createAppointment(@RequestBody Appointment appointment) {
    AppointmentResponse createdAppointment = appointmentService.createAppointment(appointment);
    return ResponseEntity.status(201).body(ApiResponse.created(createdAppointment));
}
```

**变更后**:
```java
@PostMapping
@PreAuthorize("hasRole('PATIENT')")
public ResponseEntity<ApiResponse<AppointmentResponse>> createAppointment(
        @RequestBody Appointment appointment,
        @AuthenticationPrincipal UserDetails currentUser) {
    AppointmentResponse createdAppointment = appointmentService.createAppointment(appointment, currentUser);
    return ResponseEntity.status(201).body(ApiResponse.created(createdAppointment));
}
```

**新增功能**:
- 注入当前认证用户信息 (`@AuthenticationPrincipal UserDetails currentUser`)
- 将用户身份信息传递给 Service 层进行权限验证

#### 2.2 Service 层验证增强
**文件**: `AppointmentService.java:51-96`

**新增验证逻辑**:
```java
// 验证当前认证用户的身份
User authenticatedUser = userRepository.findByUsername(currentUser.getUsername())
    .orElseThrow(() -> new BusinessException(ErrorCode.BAD_REQUEST, "当前用户不存在"));

// 确保患者只能为自己创建预约
if (!authenticatedUser.getId().equals(appointment.getPatient().getId())) {
    logger.warn("权限拒绝: 当前用户尝试为其他患者创建预约 | 当前用户ID: {}, 请求患者ID: {}",
        authenticatedUser.getId(), appointment.getPatient().getId());
    throw new BusinessException(ErrorCode.PERMISSION_DENIED, "只能为自己创建预约");
}
```

**安全增强**:
- 验证请求中的患者 ID 与当前登录用户 ID 一致
- 防止患者账户为其他患者创建预约
- 添加安全审计日志记录

#### 2.3 错误码扩展
**文件**: `ErrorCode.java:37-38`

**新增错误码**:
```java
PERMISSION_DENIED(6001, "权限不足，无法执行此操作")
```

## 变更影响分析

### ✅ 积极影响
1. **权限保障**: 确认患者角色具备完整的预约创建权限
2. **安全增强**: 防止患者账户为其他患者创建预约,避免越权操作
3. **审计完善**: 增加权限拒绝情况的日志记录,便于问题排查
4. **错误处理**: 新增专用错误码,使错误信息更加明确

### ⚪ 无影响
1. **医生角色**: 医生相关权限不受影响
2. **管理员角色**: 管理员权限保持完整
3. **其他功能**: 预约查询、更新、删除等功能不受影响

### ⚠️ 注意事项
1. **患者身份绑定**: 患者只能为自己创建预约,不能为其他人代预约
2. **JWT 认证**: 需要确保患者持有有效的 JWT 令牌
3. **数据一致性**: 请求中的患者 ID 必须与 JWT 中的用户 ID 一致
