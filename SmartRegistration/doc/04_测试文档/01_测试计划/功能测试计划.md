# 功能测试计划

## 1. 概述

本文档是智能挂号预约系统的功能测试计划，旨在确保系统的功能符合需求规格，为测试团队提供测试指导。

## 2. 测试目标

- 验证系统的所有功能是否正常工作
- 确保系统符合需求规格
- 发现并修复功能缺陷
- 确保系统的可靠性和稳定性

|------|--------|----------|

| | 注册 | 验证用户注册功能 |
| | 刷新令牌 | 验证令牌刷新功能 |

| | 科室列表 | 验证科室列表查询 |
| **医生模块** | 医生管理 | 验证医生CRUD操作 |
| | 医生详情 | 验证医生详情查询 |
| | 医生排班 | 验证医生排班查询 |

### 3.2 边界范围

- 不包括性能测试
- 不包括安全测试
- 不包括兼容性测试

## 4. 测试策略

### 4.1 测试方法

- **功能测试**：验证系统功能是否符合需求
- **集成测试**：验证模块间的集成是否正常
- **回归测试**：验证修复后是否引入新问题

### 4.2 测试环境

| 环境 | 配置 |
|------|------|
| **操作系统** | Windows 10 / Linux |
| **浏览器** | Chrome 90+ |
| **数据库** | MySQL 8.0 |
| **JDK** | OpenJDK 17 |
| **应用服务器** | Tomcat 11.0 |

### 4.3 测试工具

- **Postman**：API测试
- **JUnit**：单元测试
- **TestNG**：集成测试
- **Selenium**：UI测试

## 5. 测试用例设计

### 5.1 测试用例编号规则

```
<模块代码>-<功能代码>-<序号>
```

**模块代码**：
- AUTH：认证模块
- USER：用户模块
- DEPT：科室模块
- DOCTOR：医生模块
- SCHEDULE：排班模块
- APPOINTMENT：预约模块
- SYSTEM：系统模块

**功能代码**：
- LOGIN：登录
- REGISTER：注册
- CREATE：创建
- UPDATE：更新
- DELETE：删除
- LIST：列表查询
- DETAIL：详情查询

### 5.2 测试用例设计方法

- **等价类划分**：将输入数据划分为有效和无效等价类
- **边界值分析**：测试边界条件
- **因果图**：分析输入与输出的关系
- **错误推测**：基于经验推测可能的错误

## 6. 测试执行计划

### 6.1 测试阶段

| 阶段 | 时间 | 任务 |
|------|------|------|
| **准备阶段** | 1天 | 环境搭建、测试用例准备 |
| **执行阶段** | 3天 | 执行功能测试 |
| **回归阶段** | 1天 | 执行回归测试 |
| **报告阶段** | 1天 | 编写测试报告 |

### 6.2 测试资源

| 资源 | 数量 | 职责 |
|------|------|------|
| **测试工程师** | 2人 | 执行测试用例 |
| **开发工程师** | 1人 | 配合测试、修复bug |
| **测试环境** | 1套 | 测试执行 |

### 6.3 测试进度

| 日期 | 任务 | 负责人 |
|------|------|--------|
| Day 1 | 环境搭建、测试用例准备 | 测试团队 |
| Day 2 | 认证模块、用户模块测试 | 测试工程师A |
| Day 3 | 科室模块、医生模块测试 | 测试工程师B |
| Day 4 | 排班模块、预约模块测试 | 测试工程师A |
| Day 5 | 系统模块测试、回归测试 | 测试工程师B |
| Day 6 | 测试报告编写 | 测试团队 |

## 7. 测试标准

### 7.1 测试通过标准

- 所有功能测试用例通过率达到95%以上
- 严重和高优先级的bug都已修复
- 系统能够正常运行，无严重错误

### 7.2 测试终止标准

- 达到测试通过标准
- 测试时间到期
- 测试资源不足

## 8. 风险评估

| 风险 | 影响 | 可能性 | 缓解措施 |
|------|------|--------|----------|
| 环境搭建失败 | 测试延迟 | 低 | 提前准备环境 |
| 需求变更 | 测试范围调整 | 中 | 及时更新测试用例 |
| 资源不足 | 测试延迟 | 低 | 合理安排资源 |
| 技术难题 | 测试阻塞 | 中 | 提前沟通技术问题 |

## 9. 测试交付物

- **测试计划**：本文档
- **测试用例**：功能测试用例文档
- **测试报告**：功能测试报告
- **缺陷报告**：bug缺陷列表

## 10. 测试流程

1. **测试准备**：搭建测试环境，准备测试数据
2. **测试执行**：按照测试用例执行测试
3. **缺陷管理**：记录和跟踪缺陷
4. **回归测试**：验证缺陷修复
5. **测试报告**：编写测试报告

## 12. 总结

本功能测试计划详细说明了智能挂号预约系统的功能测试范围、策略、执行计划和标准。通过系统性的功能测试，可以确保系统的功能符合需求规格，提高系统的可靠性和稳定性。测试团队应严格按照本计划执行测试，确保测试的全面性和有效性。
