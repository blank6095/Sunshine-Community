# 智慧医院挂号预约系统 - 项目规划文档

## 1 项目概述

### 1.1 项目背景

智慧医院挂号预约系统旨在为患者提供便捷、高效的在线挂号预约服务，解决传统就医排队时间长、挂号难等问题。系统采用UniApp跨平台框架开发，支持多端部署（微信小程序、H5、iOS、Android），为用户提供统一的预约体验。

### 1.2 项目目标

- 为患者提供24小时在线预约挂号服务，减少现场排队时间
- 支持多端访问，实现一次开发多端部署
- 提供友好的用户界面和流畅的操作体验
- 确保用户数据安全和隐私保护
- 支持预约管理、医生查询、科室浏览等核心功能

### 1.3 技术栈

| 类别 | 技术选型 | 说明 |
|------|----------|------|
| 框架 | UniApp + Vue 3 | 跨平台应用开发框架 |
| 语言 | TypeScript | 类型安全的JavaScript超集 |
| 状态管理 | Pinia | 轻量级Vue 3状态管理库 |
| UI组件 | uni-ui | UniApp官方UI组件库 |
| 样式 | SCSS | CSS预处理器，支持变量和混合宏 |
| 构建工具 | Vite | 快速的现代化构建工具 |
| 代码规范 | ESLint + Prettier | 代码检查和格式化 |

### 1.4 API基础信息

| 项目 | 说明 |
|------|------|
| 基础URL | http://localhost:8080/api/v1 |
| 数据格式 | JSON |
| 认证方式 | JWT Bearer Token |
| 令牌有效期 | 24小时 |

#### 3.5.3 预约流程

```

    ↓

    调用 GET /schedules/doctor/{doctorId} 获取该医生所有排班
    ↓
Step 3: 选择时段（排班）
    调用 GET /schedules/doctor/{doctorId}/date/{date} 获取当天排班

Step 4: 选择具体时间（一个具体的时间点）
    根据排班的时间范围，生成可选时间点
    ↓
Step 5: 填写症状描述
    ↓
Step 6: 确认预约
    调用 POST /appointments 创建预约
    ↓
Step 7: 预约成功
```

#### 3.5.6 时间点生成逻辑

根据排班的startTime和endTime，以及maxPatients，生成可选时间点：

```typescript
function generateTimeSlots(schedule: Schedule): string[] {
  const slots: string[] = []
  const [startHour, startMin] = schedule.startTime.split(':').map(Number)
  const [endHour, endMin] = schedule.endTime.split(':').map(Number)

  const totalMinutes = (endHour - startHour) * 60 + (endMin - startMin)
  const interval = totalMinutes / schedule.maxPatients

  let currentMinutes = 0
  for (let i = 0; i < schedule.maxPatients; i++) {
    const hour = startHour + Math.floor((startMin + currentMinutes) / 60)
    const min = (startMin + currentMinutes) % 60
    slots.push(`${String(hour).padStart(2, '0')}:${String(min).padStart(2, '0')}:00`)
    currentMinutes += interval
  }

  return slots
}
```

#### 3.6.6 改签实现（间接支持）

```typescript

  id: number,
  newScheduleId: number,
  newAppointmentTime: string
) => {
  return request.put(`/appointments/${id}`, {
    schedule: { id: newScheduleId },
    appointmentTime: newAppointmentTime
  })
}
```

**改签流程**：
```
1. 进入改签页面
2. 选择新的排班日期
3. 调用 GET /schedules/doctor/{doctorId}/date/{newDate} 获取新排班
4. 选择新的时段
5. 确认改签
6. 调用 PUT /appointments/{id} 更新预约
```

## 5 错误处理机制

### 5.1 HTTP状态码处理

| 状态码 | 说明 | 处理方式 |
|--------|------|----------|
| 200 | 成功 | 正常处理响应数据 |
| 201 | 创建成功 | 提示用户操作成功 |
| 204 | 删除成功 | 提示用户操作成功，无需返回数据 |
| 400 | 请求参数错误 | 提示用户检查输入 |
| 401 | 未授权 | 跳转登录页 |
| 403 | 禁止访问 | 提示权限不足 |
| 404 | 资源不存在 | 提示信息不存在 |
| 500 | 服务器错误 | 提示服务繁忙 |

### 5.2 业务错误码处理

根据API文档定义的错误码：

| 错误码 | 描述 | 前端处理 |
|--------|------|----------|
| 1001 | 用户不存在 | 提示用户名或密码错误 |
| 1002 | 密码错误 | 提示密码不正确 |
| 1003 | 用户名已存在 | 提示请更换用户名 |
| 1004 | 手机号码已存在 | 提示手机号已被注册 |
| 2001 | 科室不存在 | 提示科室信息有误 |
| 3001 | 医生不存在 | 提示医生信息有误 |
| 4001 | 排班不存在 | 提示该排班已失效 |
| 4002 | 排班已取消 | 提示该排班已取消 |
| 4003 | 排班已满 | 提示该时段已约满 |
| 5001 | 预约不存在 | 提示预约信息有误 |
| 5002 | 预约已完成 | 提示该预约已完成 |
| 5003 | 预约已取消 | 提示该预约已取消 |
| 5004 | 预约时间已过 | 提示预约时间已过期 |

### 5.3 统一错误处理

```typescript
// utils/request.ts
import { getToken, clearToken } from './storage'

interface ResponseData<T = any> {
  code: number
  message: string
  data: T
}

export const request = {
  async get<T>(url: string, params?: object): Promise<T> {
    try {
      const res = await uni.request({
        url: BASE_URL + url,
        method: 'GET',
        data: params,
        header: {
          Authorization: `Bearer ${getToken()}`
        }
      })

      const data = res.data as ResponseData<T>

      if (data.code === 401) {
        clearToken()
        uni.navigateTo({ url: '/pages/login/index' })
        throw new Error('登录已过期')
      }

      if (data.code !== 200 && data.code !== 201) {
        uni.showToast({
          title: data.message || '请求失败',
          icon: 'none'
        })
        throw new Error(data.message)
      }

      return data.data
    } catch (error: any) {
      uni.showToast({
        title: error.message || '网络请求失败',
        icon: 'none'
      })
      throw error
    }
  },

  async post<T>(url: string, data?: object): Promise<T> {
    return this.request({
      url,
      method: 'POST',
      data
    })
  },

  async put<T>(url: string, data?: object): Promise<T> {
    return this.request({
      url,
      method: 'PUT',
      data
    })
  },

  async delete<T>(url: string): Promise<T> {
    return this.request({
      url,
      method: 'DELETE'
    })
  }
}
```

## 8 功能约束说明

### 8.1 API能力限制

由于API设计限制，以下功能**无法实现**：

| 功能 | 原因 | 替代方案 |
|------|------|----------|
| 密码找回 | API无此接口 | 注册新账号 |
| 就诊人管理 | 用户即患者本人 | 直接使用账户信息 |
| 头像/昵称 | 用户信息无此字段 | 使用姓名显示 |
| 消息通知 | API无此模块 | 手动刷新查看 |
| 在线支付 | API无此接口 | 线下支付（到院支付） |
| 预约改签 | 仅支持时间更新 | 取消后重新预约 |
| 用户评价 | API无此接口 | 暂无 |
| 智能推荐 | API无此功能 | 暂无 |

### 8.2 设计决策

1. **用户即患者**：系统中用户角色固定为PATIENT，登录用户即为患者本人
2. **无就诊人管理**：简化设计，患者使用自己账户预约
3. **无在线支付**：预约创建后状态为PENDING，需到医院后确认支付
4. **无消息通知**：需要用户主动刷新查看预约状态
