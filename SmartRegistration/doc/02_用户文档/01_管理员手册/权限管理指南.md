# 权限管理指南

## 1. 概述

本文档详细说明智能挂号预约系统的权限管理机制，帮助管理员了解和配置系统的权限控制。

## 2. 权限模型

系统采用基于角色的访问控制（RBAC）模型，通过以下层次进行权限管理：

- **用户**：系统的使用者
- **角色**：权限的集合
- **权限**：对资源的操作权限
- **资源**：系统中的各种实体和功能

## 3. 角色定义

### 3.1 系统角色

| 角色 | 描述 | 权限范围 |
|------|------|----------|
| **ADMIN** | 系统管理员 | 所有权限，可管理整个系统 |
| **DOCTOR** | 医生 | 可管理自己的排班和查看患者预约 |
| **PATIENT** | 患者 | 可预约和查看自己的预约记录 |

### 3.2 角色权限矩阵

| 功能 | ADMIN | DOCTOR | PATIENT |
|------|-------|--------|---------|
| 用户管理 | ✅ | ❌ | ❌ |
| 科室管理 | ✅ | ❌ | ❌ |
| 医生管理 | ✅ | ❌ | ❌ |
| 排班管理 | ✅ | ✅ (自己) | ❌ |
| 预约管理 | ✅ | ✅ (自己) | ✅ (自己) |
| 系统配置 | ✅ | ❌ | ❌ |
| 系统监控 | ✅ | ❌ | ❌ |

## 4. 权限配置

### 4.1 用户角色分配

#### 4.1.1 创建用户时分配角色

- **接口**：`POST /users`
- **请求体**：
  ```json
  {
    "username": "doctor1",
    "password": "password",
    "name": "张医生",
    "role": "DOCTOR",
    "status": "ACTIVE"
  }
  ```

#### 4.1.2 更新用户角色

- **接口**：`PUT /users/{id}`
- **请求体**：
  ```json
  {
    "role": "ADMIN"
  }
  ```

### 4.2 权限控制实现

系统通过以下方式实现权限控制：

1. **URL路径控制**：在SecurityConfig中配置
2. **方法级权限**：使用@PreAuthorize注解
3. **业务逻辑控制**：在服务层实现数据访问控制

#### 4.2.1 URL路径控制示例

```java
@Configuration
@EnableWebSecurity
@EnableMethodSecurity(prePostEnabled = true)
public class SecurityConfig {
    @Bean
    public SecurityFilterChain securityFilterChain(HttpSecurity http) throws Exception {
        http
            .authorizeHttpRequests(auth -> auth
                .requestMatchers("/auth/**", "/health/**").permitAll()
                .requestMatchers("/admin/**").hasRole("ADMIN")
                .anyRequest().authenticated()
            );
        return http.build();
    }
}
```

#### 4.2.2 方法级权限示例

```java
@RestController
@RequestMapping("/users")
public class UserController {
    @PreAuthorize("hasRole('ADMIN')")
    @PostMapping
    public ResponseEntity<ApiResponse<User>> createUser(@RequestBody User user) {
        // 实现逻辑
    }
}
```

## 5. 权限管理操作

### 5.1 查看用户角色

- **接口**：`GET /users/{id}`
- **响应**：用户信息，包含角色

### 5.2 批量分配角色

- **接口**：`PUT /users/batch-role`
- **请求体**：
  ```json
  {
    "userIds": [2, 3, 4],
    "role": "DOCTOR"
  }
  ```

### 5.3 角色权限查询

- **接口**：`GET /admin/roles`
- **响应**：角色列表及权限

## 6. 权限审计

### 6.1 权限变更日志

系统会记录所有权限变更操作：
- 用户角色变更
- 权限配置修改
- 敏感操作执行

### 6.2 审计查询

- **接口**：`GET /admin/audit/logs`
- **参数**：
  - `type`：操作类型
  - `startDate`：开始日期
  - `endDate`：结束日期
- **响应**：审计日志列表

## 7. 权限最佳实践

### 7.1 最小权限原则

- 只授予用户完成任务所需的最小权限
- 避免过度授权

### 7.2 权限分离

- 管理权限与操作权限分离
- 敏感操作需要多重授权

### 7.3 定期权限审查

- 定期审查用户权限
- 及时回收不需要的权限

### 7.4 权限继承

- 利用角色继承简化权限管理
- 建立清晰的权限层级

## 8. 常见权限问题

### 8.1 权限不足

- **问题**：用户尝试访问无权限的资源
  **解决**：检查用户角色，确保有相应权限

### 8.2 权限冲突

- **问题**：权限配置冲突
  **解决**：检查权限配置，确保权限规则正确

### 8.3 权限泄露

- **问题**：用户获取了不应有的权限
  **解决**：检查权限配置，及时修正

## 9. 权限管理流程

### 9.1 新用户权限分配流程

1. 创建用户账号
2. 分配适当的角色
3. 根据需要调整具体权限
4. 通知用户权限变更

### 9.2 权限变更流程

1. 提出权限变更申请
2. 管理员审核
3. 执行权限变更
4. 记录变更日志
5. 通知相关用户

### 9.3 权限回收流程

1. 识别需要回收的权限
2. 执行权限回收
3. 记录回收日志
4. 通知用户权限变更

## 10. 安全注意事项

- 定期更新管理员密码
- 不要共享管理员账号
- 敏感操作需双人确认
- 定期备份权限配置
