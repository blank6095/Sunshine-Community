错误处理代码示例：
```typescript
const http = {
  get: async (url, params) => {
    try {
      // 请求逻辑
    } catch (error) {
      // 错误提示
      uni.showToast({ title: getErrorMessage(error.code), icon: 'none' })
    }
  }
}
```

## 四、安全考虑

### 4.1 安全措施

| 安全项 | 状态 | 说明 |
|-------|------|------|
| Token 认证 | ✅ 完整实现 | Bearer Token + 本地存储 |
| Token 刷新 | ✅ 预留接口 | API 支持 |
| XSS 防护 | ✅ 使用 v-html 受限 | 遵循 Vue 安全建议 |
| 敏感数据处理 | ✅ 不存储密码明文 | 仅存储 Token 和用户信息 |
| 权限检查 | ✅ 路由级权限 | 未登录跳转登录页 |

### 4.2 安全最佳实践

- 所有用户输入在前端进行验证
- API 请求统一使用 Token 认证
- 敏感操作有二次确认（如取消预约）
- 使用 HTTPS（生产环境强制）

## 五、页面功能详解

### 5.1 首页 (pages/index/index.vue)
- 功能：科室选择入口、快捷导航、我的预约入口
- 特点：蓝色医疗风格主题，清晰的功能区域

### 5.2 登录/注册 (pages/login/)
- 登录：用户名+密码，完整表单验证
- 注册：完整用户信息收集，性别选择
- 特点：安全的登录流程，状态持久化

### 5.3 科室模块 (pages/department/)
- 列表：科室列表展示
- 详情：科室信息展示
- 特点：清晰的科室分类

### 5.4 预约记录 (pages/record/)
- 列表：按状态筛选（全部、待确认、已确认、已完成）

- 取消：预约取消功能（带确认）
- 特点：Tab 切换，状态标签

### 5.5 个人中心 (pages/user/)
- 功能：用户信息展示、个人资料修改
- 特点：简洁的个人信息展示
